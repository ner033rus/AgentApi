# AgentAPI — сервер метрик

Сервис на **Python / FastAPI**, который собирает снимок состояния хоста и отдаёт его по **REST** и по **WebSocket** при заметных изменениях.

---

## Зачем это нужно

| Цель | Как |
|------|-----|
| Смотреть нагрузку с другой машины или из UI | JSON по HTTP |
| Стримить обновления без постоянного опроса | WebSocket `/api/v1/stream` |
| Безопасно гасить только «ollama»-процессы | `DELETE /api/v1/processes/{pid}` |

Метрики рассчитываются **на той машине, где запущен сервер** (это не клиентский мониторинг).

---

## Что собирается

| Область | Содержимое |
|---------|------------|
| **CPU** | Загрузка, % |
| **RAM** | Использование, %; used/total в байтах |
| **GPU** | Через `nvidia-smi` (NVIDIA): util, память, температура, питание и т.д. Если `nvidia-smi` нет — блок помечается как недоступный |
| **Ollama** | Список процессов, в имени/cmdline которых есть подстроки из настроек (по умолчанию `ollama`) |

Интерактивная документация OpenAPI: **`/docs`** после запуска.

---

## Требования

- **Python 3.10+** (рекомендуется 3.11)
- Для GPU-метрик: **NVIDIA** драйвер и **`nvidia-smi`** в `PATH` на сервере

## Установка и запуск

### Вариант 1: скрипт `run.sh` (venv + зависимости + uvicorn)

```bash
cd Server
chmod +x run.sh
./run.sh
```

Скрипт создаст `.venv`, установит `requirements.txt` и запустит приложение.

### Вариант 2: вручную

```bash
cd Server
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
export PYTHONPATH="$PWD"
uvicorn app.main:app --host 0.0.0.0 --port 8765 --app-dir .
```

## Переменные окружения

Префикс **`AGENT_API_`** (см. `app/config.py`).

| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `AGENT_API_HOST` | `0.0.0.0` | Хост при запуске через `run.sh` |
| `AGENT_API_PORT` | `8765` | Порт |
| `AGENT_API_LOG_LEVEL` | `info` | Уровень логов |

Дополнительно можно положить файл **`.env`** в каталог `Server` (тот же префикс).

## Основные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/health` | Проверка живости |
| GET | `/` | Подсказка по ссылкам |
| GET | `/api/v1/metrics` | Полный снимок; опционально `?include=cpu,ram,gpu,ollama` |
| GET | `/api/v1/metrics/cpu` и др. | Отдельные блоки |
| WebSocket | `/api/v1/stream` | Первое сообщение клиента — JSON `subscribe` с полем `watch` |
| DELETE | `/api/v1/processes/{pid}` | SIGTERM только для процессов, распознанных как Ollama |

Пример подписки на поток (после `accept` отправить текстом):

```json
{"action":"subscribe","watch":["cpu","ram","gpu","ollama"],"poll_interval_sec":1.0}
```

## Сервис под Linux (опционально)

В репозитории есть заготовки:

- `install-debian-service.sh` — установка systemd unit
- `agent-api.service` — unit-файл

Смотрите комментарии внутри скриптов и при необходимости поправьте пути под свою систему.
